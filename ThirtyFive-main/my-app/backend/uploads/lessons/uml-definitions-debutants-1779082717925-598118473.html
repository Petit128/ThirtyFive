Diagrammes structurels — "comment c'est construit"
Classes

Diagramme de classes

Représente les "modèles" du système : les types d'objets, leurs caractéristiques et leurs relations.

Composants clés : Classe · Attribut · Méthode · Héritage · Association · Multiplicité (1..* etc.)

Mots-clés : extends · implements · <> · composition · agrégation

Objets

Diagramme d'objets

Photographie d'un instant précis : montre des instances concrètes (objets réels) d'un diagramme de classes.

Composants clés : Objet (instance) · Valeur d'attribut · Lien (instance d'association)

Mots-clés : :NomClasse · snapshot · instanciation

Composants

Diagramme de composants

Montre les grands blocs logiciels du système (modules) et comment ils s'utilisent les uns les autres.

Composants clés : Composant · Interface fournie · Interface requise · Dépendance

Mots-clés : <> · port · connecteur · dépendance

Déploiement

Diagramme de déploiement

Montre où le logiciel s'exécute physiquement : serveurs, machines, réseaux, appareils.

Composants clés : Nœud (serveur/machine) · Artefact (fichier .jar, .exe…) · Lien réseau

Mots-clés : <> · <> · déploiement · topologie

Diagrammes comportementaux — "comment ça se passe"
Activité

Diagramme d'activité (Flowchart)

Modélise un processus étape par étape. Comme un organigramme, avec la possibilité de faire des tâches en même temps.

Composants clés : Action (rectangle arrondi) · Décision (losange) · Fork (barre de séparation) · Join (barre de fusion) · Début (cercle plein) · Fin (cercle cerclé) · Couloirs (swimlanes)

Mots-clés : flux de contrôle · activités parallèles · condition garde · branche

Séquence

Diagramme de séquence

Montre les messages échangés entre les objets dans l'ordre chronologique (de haut en bas).

Composants clés : Ligne de vie (lifeline) · Message (flèche) · Barre d'activation · Fragment combiné (loop, alt, opt)

Mots-clés : appel synchrone · réponse · activation · interaction

États

Diagramme d'états

Décrit le cycle de vie d'un objet : les états qu'il traverse et les événements qui le font changer d'état.

Composants clés : État (rectangle arrondi) · Transition (flèche) · État initial · État final · Événement déclencheur · Garde (condition)

Mots-clés : transition · trigger · état composite · machine à états

Temps

Diagramme de temps (Timing)

Représente les changements d'état sur un axe temporel. Idéal pour les systèmes industriels ou embarqués.

Composants clés : Axe X (temps) · Ligne de vie (état de l'objet) · Changement d'état (ligne verticale) · Contrainte de durée · Contrainte de temps

Mots-clés : timeline · durée · synchronisation · contrainte temporelle · chronogramme
